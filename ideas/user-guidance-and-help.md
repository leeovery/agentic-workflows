# User Guidance & Help Along the Way

## The Idea

Add better explanations and contextual help throughout the workflow so users understand what's happening and what to expect at each stage.

## Context

The workflows produce amazing results and move through solid discussion, planning, and implementation strategies. But the user experience could be improved by making the process more transparent — explaining *why* each phase matters, what's about to happen, and what the user should be thinking about at each point.

## Possible Directions

- Brief contextual explanations at phase transitions ("You're entering specification — this phase synthesises your discussions into a validated spec. Expect questions about gaps and contradictions.")
- Inline help for menu options and prompts so users understand what each choice means before committing
- Progress indicators showing where the user is in the overall pipeline and what's ahead
- First-run guidance that's more detailed, tapering off for experienced users
- **Phase previews** at each transition — what the phase does, what it'll ask, what your role is, what the output will be. Particularly useful for new users or returning after a break. Should be suppressible (project-level setting or tapers off after first completion of each phase).
